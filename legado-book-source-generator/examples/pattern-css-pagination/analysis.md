# 网站分析

## 搜索

- 页面入口或触发方式: 站点搜索页 `search.php`
- 请求链路或接口来源: GET `https://www.163zw.com/search.php?q={{key}}&p={{page}}`
- 稳定抓取依据: 搜索结果列表位于 `div.box.hot > div.row > div.col-12.col-md-6`
- 风险点: 搜索结果中的书名包含方括号前缀，需要净化
- Legado 规则建议: 使用 CSS 选择器抽取列表；书名字段追加正则净化；详情链接与封面链接补全根域

## 详情

- 页面入口或触发方式: 搜索结果或目录页跳转到书籍详情页
- 请求链路或接口来源: 书籍详情 HTML 页面
- 稳定抓取依据: 书名、作者、封面、简介位于稳定详情模块 `div.row.book_info`
- 风险点: 简介尾部存在固定宣传文本，需要净化
- Legado 规则建议: 直接使用 CSS 选择器抽取；简介使用正则删除前缀和尾部宣传文本；目录与详情页同源，因此 `tocUrl` 留空

## 目录

- 页面入口或触发方式: 详情页直出目录
- 请求链路或接口来源: 详情页内目录模块与目录分页链接
- 稳定抓取依据: 章节列表位于 `div.book_list.book_list2 ul.row li`
- 风险点: 目录存在分页；目录下一页链接仅在分页目录页出现
- Legado 规则建议: 使用 `nextTocUrl` 识别分页链接；章节链接补全根域

## 正文

- 页面入口或触发方式: 章节详情页
- 请求链路或接口来源: 正文 HTML 页面
- 稳定抓取依据: 正文主体位于 `article.font_max` 容器
- 风险点: 单章多页；翻页链接与下一章链接共存，需要显式区分
- Legado 规则建议: 正文用 JS 清洗 HTML、分页标记和多余空行；`nextContentUrl` 仅在同章分页时返回下一页 URL
